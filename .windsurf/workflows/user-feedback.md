## 🔄 用戶回饋收集流程

### 觸發時機
**必須在以下情況使用 mcp-feedback-collector：**
- ✅ **任務完成前** - 所有工作即將結束時
- ✅ **工作總結時** - 提供完成報告後
- ✅ **提問前** - 需要用戶確認或澄清時
- ✅ **重要決策前** - 影響專案方向的選擇時

### 執行步驟
1. **準備回饋內容** - 簡潔描述當前工作狀態
2. **呼叫工具** - 使用 `mcp2_collect_feedback`
3. **等待回饋** - 檢查用戶回饋內容
4. **持續收集** - 直到回饋為空才結束

### 回饋內容建議
- 📝 **工作進度** - 完成百分比和主要成果
- 🎯 **問題點** - 遇到的困難或需要協助
- ❓ **需要確認** - 待用戶決定的事項
- 📋 **下一步** - 建議的後續行動

### 注意事項
- 🚫 **不要跳過** - 即使沒有問題也要收集回饋
- 🔄 **重複收集** - 持續直到用戶無回饋為止
- 📊 **記錄結果** - 用於改善後續工作品質
